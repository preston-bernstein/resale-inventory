## Notes
No `npm run migrate` script exists — migrations in `data/migrations/` apply
automatically via `lib/db.ts`'s `VERSIONED_MIGRATIONS` array on first DB
access (module init), gated by `PRAGMA user_version`. Verify migrations by
running the app/tests against a scratch DB (`BOOKSELLER_DB_PATH`), not a
dedicated migrate command.

This feature directly extends the pattern shipped in PR #12 (`clothing_brands`
table, `lib/brands.ts`, `BrandCombobox.tsx`, `app/api/brands/route.ts`) — read
those files as the concrete precedent before implementing any step.
